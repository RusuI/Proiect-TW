Introduction

Scope

The product whose specifications are presented in this document consists of a web application which generates recipes based on the filtering the ingredients chosen by the user, depending on other specifications such as cooking time, difficulty, unwanted ingredients.

Currently, the application is in its starting phase, which is why this is the first version of use. The purpose of the application is to provide its users with a simple and efficient way to search, save, and create various recipes based on the selected criteria.

Documents conventions

The main convention for making this document is to use the Scholarly HTML template. [1]

The intended audience

The present document is addressed mainly to the teachers responsible for the evaluation of the project within the Web Technology discipline. By expanding the area, it can be transformed into a document addressed to potential users, or even investors who would like to use the application in the long run.

Aplication domain

The “WhaF” web application described in this document aims to make it easier to search for specific recipes for its users.

One of the great advantages of this application is the efficiency of filtering recipes and reducing the search time available to the user, as they can choose the specifications they want for the recipes they are looking for.

Basically, the main goal is for the user to use their time to prepare a fixed recipe based on the ingredients they want, instead of looking for one online. Regarding possible bussines strategies, the application it could be an intermediary between brands and suppliers of certain foods, utensils, restaurants relevant to the content they are looking for.

General description

Product perspective

The WhaF project is a new, individual product, appeared in the context of the evaluation of the Web Technologies discipline. The idea was chosen after consulting the list of projects provided by the laboratory coordinator, Amaria Ciprian.

Basically, our project comes with an innovative idea and as a replacement for the already existing recipe search services, such as culinary blogs, websites that offer as a content a suite of recipes, offering methods more effective based on recommendations.

System features

  • The ability to create an account is one of the main features of the system, as it offers the user additional features (the option to add a recipe, a list of favorites,etc.). Information about a user who creates their account is saved in a database server, while the preferences of the last search of an unauthenticated user are saved at the browser level. The registration of a new user into the system will lead to the population of the database.
  • In a user's profile page ,the ability of changing account settings will change certain data on the database server and will provide the user with the option to update their informations anytime they want.
  • Recipe Search : This feature has top priority, representing the essence of the application. The user (logged in or not logged in) can select the ingredients they want and those that they do not like, the level of difficulty, the time food preparation and cooking and number of servings. In order for the application to work as fast as possible, Ajax will be used, so it will not reload the page, but only add or delete new recipes that meet the criteria. The preferences of the last search will be saved at the browser level, with the help of cookies.
  • Popular : The user (logged in / logged out) can access this page to see the most popular recipes / ingredients and the most unwanted ingredients. Popular recipes will be identified by the number of people who prepared the recipe. Popular / unpopular ingredients will be determined by the number of times it was marked as desirable or undesirable in the search operation. It will also be possible to download the rankings in HTML, CSV and JSON format.
  • Posting a photo with your own preparation : The functionality of posting a photo with the product has a high priority, as the popularity of a recipe is determined by the number of photos uploaded by users on the page of a recipe.
  • Send a question : The priority of this feature is average. It comes as a service addressed to users who have questions about the web page or any type of problem reported.
  • When modifying the list of favorites (adding / deleting the recipe) Ajax will be used, as in the case of searching for a recipe , for the fast, efficient modifications of the respective page.
  • Add recipe : Probably one of the most interesting features,this functionality provides the user with the ability to add their own recipe that will be available on the site for other users to prepare. The users who chooses to add a recipe will be able to select from numerous ingredients,categories,times of preparatios etc.
  • After adding a recipe, the user will be provided with the ability to see their own recipes loaded on the website by going to the My Recipes option available on their profile page.

Technical specifications and technologies

As software components and additional services for the "WhaF" application, we will use:

  1. HTML : The markup language that is used to display every document of the app on the browser,beging used for the front-end side of the application;
  2. CSS : The style sheet language used to display the application content in a user-friendly manner;
  3. JavaScript : Used in the big majority of our application, especially;
  4. XAMPP for Windows 8.0.3. - free software, open-source and cross-platform web server software package containing an HTTP server (respectively Apache), a MySQL database and an interpreter for scripts written in PHP . [2]
  5. AOS (Animate On Scroll Library) - a scrolling animation library used in CSS3, useful for animating any element of the website during scrolling. In order to use this library, stylesheet along with JS files, using a CDN. After initializing the library, using the specific function AOS.init (), we need to add the specific attributes. To use basic animations just add data-aos = "animation_name" to your HTML elements. There are several types of animations to choose from. For example, you can add blur animations such as "fade", "fade-up" and "fade-down-left". Similarly, you can add flip and slide animations as well they would be "flip-up", "flip-left", "slide-down" and "slide-right". [3][5]
  6. AJAX – used on the client-side to create asynchronous web applications, AJAX is a set of web development techniques using many web technologies . With Ajax, web applications can send and retrieve data from a server asynchronously without interfering with the display and behaviour of the existing page. In WhaF web application, AJAX is used for searching the ingredients,adding a recipe, adding and deleting to favorites,etc. [4]
  7. To save the last searches of users, cookies will be used to store their data at the browser level.
  8. For rankings related to preferences, the most popular ingredients / recipes (or the most avoided) the web application will display rankings in HTML, CSV and JSON formats.
  9. As the main programming language, the application uses PHP, for managing the communication with the database,tracking user sessions and other verifications that require a scripting language.
  10. SQL statements are used to manipulate data from the database.
  11. MySQL is the DBMS used by the application to store and manage every data that the application uses.Our tables are:
    1. "faq" - it stores the data related to the questions about the application;
    2. "favorites" - keeps the relationship between a user and their favorite recipes;
    3. "ingredients" - a table with all the ingredients available for the available recipes;
    4. "recipe" - stores all the recipes available on the website;
    5. "recipe_pictures" - stores all the pictures for a recipe;
    6. "recipe_ingr" - for every recipe, it keeps a list with all the ingredients of that recipe;
    7. "votes" - keeps the track of the score of every recipe;
  12. The communication protocol for the network server used is the HTTP protocol.
